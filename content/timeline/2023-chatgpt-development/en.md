---
title: "Started using ChatGPT in development"
summary: "Early AI coding experience: a simple chat interface, manual code copying, and the first lessons about context."
---

In 2023, I started using ChatGPT in software development. At that time it was mostly a chat interface with GPT-3.5: no proper IDE integration, no project connectors, no file access, and no direct tool use. Code appeared inside the chat response, and I had to copy it, paste it, check it, and manually adapt it to the real project.

As a code-writing tool, it often worked poorly. The model could make mistakes, lose context, suggest the wrong approach, or produce code that looked convincing but still needed careful review. But even then, ChatGPT was useful for explaining code, breaking down unfamiliar concepts, and turning confusing errors into concrete questions.

At the same time, I was not fully sure how much AI should be used while learning. On one side, it helped me understand code and development faster. On the other side, I could clearly see the risk: relying on it too much could slow down the ability to write code independently. Still, even then it felt less like a temporary toy and more like a direction that would keep developing quickly. AI was already helping me think not only about individual lines of code, but also about architecture, relationships between parts of a project, and the overall approach to a task.

This experience matters because it gave me an early understanding of how AI products for development evolve. I started to see why context is critical: which files and requirements the model can see, how much information fits into a prompt, how token limits affect the work, and why good output depends not only on the model but also on the way the task is framed.

Now, after several years of practicing with AI-assisted development tools, that early experience helps me use them more effectively. I better understand when AI speeds up the work, when it needs boundaries, how to verify the result, and why responsibility for the code always stays with the developer.
